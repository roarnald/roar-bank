# Context

Upcoming penetration testing for the National Lottery platform required the team to assess its security posture. Ronald, as Solutions Architect, self-initiated a security review exercise rather than waiting for external testers to discover vulnerabilities.

# Problem

No systematic security review process existed for the NL platform. Relying solely on an external penetration test would surface issues late and reactively, with no internal baseline.

# Actions

- Adapted the **OWASP Web Application Security Checklist** to create a business-specific security checklist for the NL platform.
- Conducted the checklist review against the platform's architecture and implementations.
- **Identified approximately 10 areas of improvement** for the development team to patch.
- Communicated findings to the development team and ensured remediation was prioritized.

# Impact

- Improved security posture before the penetration test — reducing the number of findings that external testers would discover.
- Gave the team *"better confidence moving into the upcoming round of penetration testing."* (2025 Appraisal)
- Established a repeatable security review process that can be used before future penetration tests or compliance audits.
- Demonstrates proactive risk management as part of the SA role, beyond technical design.
