# Context

Lotto6D KJW needed a central NodeJS API server microservice to handle simple API requirements and lift the reliance on an overseas team for custom feature development. This was also a vehicle to extend full-stack capability within the Frontend team.

# Problem

- The team needed a NodeJS microservice but no developer on the team had built one in production before (except Ronald).
- Reliance on an overseas team was a bottleneck for custom feature delivery.
- The overseas team dependency reduced agility.

# Actions

- Placed developer Jacky in the lead role to build the microservice.
- Broke down the feature into **bite-sized actionable steps** for Jacky to follow and study independently.
- Provided working samples from Ronald's prior fullstack experience (138DHW).
- Set Jacky up for a smooth delivery process, including deployment procedures.
- Collaborated to resolve multiple road blockers together during development.
- Allowed Jacky to own the delivery — did not take over or solve problems for him.

# Impact

- Jacky successfully built and deployed a **production NodeJS microservice** — his first time in the domain.
- **Lifted reliance on the overseas team** — team could now build custom server-side features independently.
- Jacky shared his lessons during Frontend Sharing, publicly crediting Ronald.
- *"I felt that this provided a huge sense of accomplishment for Jacky as he was able to pick up and deliver in a domain that he has never stepped foot into."* (Ronald, 2025 Appraisal)
- Extended full-stack capability to a second developer in the team.
