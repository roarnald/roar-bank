# Summary

A NodeJS microservice acting as a central API hub for Lotto6D KJW — replacing reliance on an overseas team for custom server-side features. This was Jacky's first production server-side project, with Ronald as architect and mentor.

# Architecture

- NodeJS microservice with file-based routing (following Ronald's NodeJS Server Template Foundation)
- Dockerised for Kubernetes deployment
- Custom API endpoints replacing overseas team dependencies

# Business Problem

Lotto6D KJW required custom server-side functionality that the team previously had to request from an overseas team. This created dependency, slow turnaround, and limited customisation ability.

# Outcomes

- Production NodeJS microservice delivered by Jacky (first time in the domain)
- Overseas team dependency lifted for this product's server-side features
- Jacky presented learnings at Frontend Sharing, publicly crediting Ronald
- Extended full-stack capability to a second developer in the team

# Lessons Learned

- Structured step-by-step guidance with working reference samples enables a developer to deliver in an unfamiliar domain.
- Allowing the developer to own the solution (not taking over) builds lasting capability.
- The NodeJS Server Templates created from New138 (2024) had immediate, practical payoff.
